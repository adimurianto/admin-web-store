# Project Plan: Admin Web Store

## Tech Stack
- **Framework:** CodeIgniter 4
- **CSS Framework:** Bootstrap 5
- **Database:** MySQL

## 1. Database Schema

### Tabel `users`
- `id`: VARCHAR/UUID (Primary Key, Generated by System)
- `nama`: VARCHAR
- `jabatan`: ENUM('Admin', 'Gudang', 'Guest')
- `email`: VARCHAR
- `username`: VARCHAR (Unique)
- `password`: VARCHAR (Hashed)

### Tabel `categories`
- `id`: VARCHAR/UUID (Primary Key, Generated by System)
- `nama_kategori`: VARCHAR
- `foto`: VARCHAR (File path/name)
- `parent_id`: VARCHAR/UUID (Nullable, Self-referencing Foreign Key)
- `edited_by`: VARCHAR/UUID (Foreign Key ke `users.id`)
- `last_edited`: DATETIME

### Tabel `items`
- `id`: VARCHAR/UUID (Primary Key, Generated by System)
- `nama_item`: VARCHAR
- `deskripsi_item`: TEXT
- `foto`: VARCHAR (File path/name)
- `category_id`: VARCHAR/UUID (Foreign Key ke `categories.id`)
- `harga`: INT/DECIMAL
- `edited_by`: VARCHAR/UUID (Foreign Key ke `users.id`)
- `last_edited`: DATETIME

---

## 2. Hak Akses (Role Base Access Control)
Sistem menggunakan Middleware / Filters di CodeIgniter 4 untuk membatasi rute (URL) berdasarkan session `jabatan`.

| Menu / Fitur | Admin | Gudang | Guest |
| :--- | :---: | :---: | :---: |
| **Login / Logout** | ✅ | ✅ | ✅ |
| **Dashboard** | ✅ (Lihat) | ✅ (Lihat) | ✅ (Lihat) |
| **Manajemen Item** | ✅ (Lihat, Tambah, Edit) | ✅ (Tambah, Edit) | ✅ (Hanya Lihat) |
| **Kategori Item** | ✅ (Lihat, Tambah, Edit) | ✅ (Lihat) | ✅ (Hanya Lihat) |
| **Users Admin** | ✅ (Lihat, Tambah, Edit) | ❌ (Tidak ada akses) | ❌ (Tidak ada akses) |

*Catatan untuk Gudang:* Hanya bisa melakukan Create / Update di "Manajemen Item". Di menu lain (Dashboard & Kategori) hanya Read-only (tombol aksi disembunyikan). Menu "Users Admin" diblokir.
*Catatan untuk Guest:* Hanya memiliki akses Read-only untuk menu Dashboard, Manajemen Item, dan Kategori Item. Tidak bisa melakukan aksi Tambah maupun Edit. Menu "Users Admin" diblokir.

## 3. Detail Fitur & Spesifikasi Form

### A. Login & Logout
- **Login:** Form autentikasi yang menggunakan `username` dan `password` (proses validasi menggunakan `password_verify`).
- **Logout:** Menghancurkan session user dan mengarahkan kembali ke halaman login.

### B. Dashboard
- Menampilkan ringkasan data statistik dalam bentuk Card:
  1. Jumlah Total Item
  2. Jumlah Total Kategori
  3. Jumlah Total User Admin (dengan filter jabatan = 'Admin')

### C. Manajemen Item
- **List Data:** Menampilkan semua item di dalam tabel dengan fitur DataTables. Terdapat tombol "New Item" dan "Edit" yang dirender sesuai dengan hak akses user.
- **Form Item (Create/Edit):**
  - **ID unik:** Digenerate oleh sistem di backend saat proses insert data (menggunakan UUID atau format ID khusus).
  - **Nama Item:** Input manual (Text).
  - **Deskripsi Item:** Input manual (Textarea).
  - **Upload Foto:** Input file. Akan ditambahkan script Javascript (menggunakan `FileReader`) untuk menampilkan live preview foto secara langsung ketika file dipilih, sebelum proses upload (submit form).
  - **Kategori Barang:** Input select dropdown yang list datanya diload secara dinamis dari tabel `categories`.
  - **Harga:** Input manual (Text). Akan ditambahkan Javascript event listener (atau library seperti AutoNumeric) untuk memberikan format ribuan saat diketik (contoh: `1000000` menjadi `1.000.000`). Data akan dibersihkan dari titik/koma sebelum masuk ke database.
  - **Edited by:** Diisi otomatis di backend mengambil ID/Nama user dari session admin yang sedang aktif login.
  - **Last edited:** Diisi otomatis di backend. Format standar ketika ditampilkan ke user adalah `dd/mm/YYYY H:i:s` (contoh: `22/04/2026 12:55:01`).

### D. Kategori Item
- **List Data:** Menampilkan semua kategori yang telah ditambahkan. Terdapat tombol "New Kategori" dan "Edit" sesuai hak akses.
- **Form Kategori (Create/Edit):**
  - **ID unik:** Digenerate oleh sistem di backend.
  - **Nama Kategori:** Input manual (Text).
  - **Upload Foto:** Input file lengkap dengan fitur live preview foto menggunakan Javascript sebelum submit.
  - **Parent Kategori:** Input select dropdown berisikan kategori-kategori yang sudah ada. Digunakan jika ingin menjadikan kategori yang dibuat sebagai subkategori (nullable jika ia adalah parent/main kategori).
  - **Edited by:** Diisi otomatis di backend sesuai admin yang sedang login.
  - **Last edited:** Diisi otomatis dengan format `dd/mm/YYYY H:i:s`.

### E. Users Admin
- **List Data:** Menampilkan daftar user admin. Hanya role Admin yang dapat mengakses menu ini. Terdapat tombol "New User" dan "Edit".
- **Form User (Create/Edit):**
  - **ID unik:** Digenerate otomatis oleh sistem.
  - **Nama User:** Input manual (Text).
  - **Jabatan:** Input select dropdown dengan pilihan baku (Admin, Gudang, Guest).
  - **Email:** Input manual (Email).
  - **Username:** Input manual (Text). Digunakan untuk akses login.
  - **Password:** Input manual (Password). Saat disimpan, password akan dienkripsi dengan metode hashing menggunakan fungsi `password_hash()` pada CodeIgniter/PHP.

## 4. Langkah Pengerjaan
1. **Setup Project CI4 & Database:** Konfigurasi `.env` untuk koneksi MySQL. Buat dan jalankan *Migrations* untuk tabel `users`, `categories`, dan `items`.
2. **Setup Template Admin:** Implementasi layout dengan Bootstrap 5 (Sidebar, Topbar, Content).
3. **Modul Auth & Roles:** Pembuatan Controller Login, konfigurasi session, dan pembuatan *Filters* (Middleware) di CodeIgniter untuk mengamankan route sesuai spesifikasi tabel hak akses.
4. **Modul Master Data:** Pembuatan CRUD untuk *Kategori Item*, *Manajemen Item*, dan *Users Admin*. Sertakan juga implementasi logic Javascript (Format Rupiah & Live Preview Image).
5. **Modul Dashboard:** Mengambil summary data via model dan mendisplay angka statistik di halaman index Dashboard.
6. **Testing:** Melakukan uji coba menyeluruh untuk memastikan upload foto berjalan normal, input form tersimpan dengan tepat, password terenkripsi, dan hak akses user (Admin, Gudang, Guest) bekerja tanpa kebocoran akses.
